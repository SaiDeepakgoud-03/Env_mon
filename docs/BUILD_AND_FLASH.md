# Build And Flash

## Install ESP-IDF

Use an ESP-IDF shell where `idf.py` is on PATH.

```powershell
idf.py --version
```

## Certificates

The firmware uses Fleet Provisioning by claim. Place these files in `main/`:

```text
AmazonRootCA1.pem
claim.pem.crt
claim.private.pem.key
```

The claim certificate must have a tightly scoped claim policy that only allows connection and Fleet Provisioning topics. Per-device certificates are generated by AWS IoT Core and stored by the device in NVS.

## Build

```powershell
idf.py set-target esp32
idf.py build
```

## Flash

```powershell
idf.py -p COM5 flash monitor
```

## First Boot

1. Device starts `EnvMon-Setup-XXXX` hotspot.
2. Connect from phone or laptop.
3. Captive portal opens at `http://192.168.4.1`.
4. Enter Device ID, Wi-Fi credentials and location.
5. Device reboots, connects to Wi-Fi and starts AWS onboarding.

Hold BOOT for more than five seconds during reset to wipe NVS and return to setup mode.
